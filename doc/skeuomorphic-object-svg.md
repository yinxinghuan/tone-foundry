# Tone Foundry — 拟物化小型物件 SVG 绘制基准

这份基准适用于游戏中的任何小型真实物件：效果器、拾音器、琴桥、旋钮、开关、磁带机、踏板、仪表、收音机、工具、机械零件或超现实电子器件。目标不是“加上渐变看起来像 3D”，而是在有限 SVG 尺寸内建立可相信的**结构、材料、功能与尺度**。

## 1. 先回答它是什么，才开始画

每个物件先写一张 `Object Card`，至少确定：

| 项目 | 必须回答的问题 |
| --- | --- |
| 功能 | 它做什么？玩家从哪里操作？ |
| 原型 | 它属于哪一类真实物件：冲压钢壳、铸造盒体、木质器件、工具、仪表、玻璃电子件？ |
| 关键轮廓 | 去色后最能识别它的外形是什么？ |
| 接触点 | 手指、脚、弦、线缆或其他物件在哪里碰到它？ |
| 装配方式 | 外壳如何开合、面板如何固定、零件为什么在这里？ |
| 材料 | 每一面分别是喷漆金属、拉丝金属、塑料、橡胶、木、玻璃还是纸质标签？ |
| 状态 | 关闭 / 激活 / 受力 / 磨损时，哪一个局部发生变化？ |

如果两个物件只靠颜色区分，说明原型还没有建立。至少让它们在轮廓、控制布局、装配关系三项中有两项不同。

## 2. 真实感来自装配逻辑，不来自复杂特效

从后到前按这个顺序搭建：

1. **轮廓和厚度**：先画正面与可见侧面；尖角处的侧面必须连续，不能裂成两片。圆角、倒角、折边和侧壁厚度要解释物件如何成型。
2. **主体材料**：每一块材料只使用一套克制的明暗逻辑。喷漆金属是方向性亮暗；拉丝金属是低对比细纹；塑料是柔和高光；木材沿纹理变化；玻璃只出现边缘高光和内部暗层。
3. **可拆部件**：面板、护板、旋钮、螺丝、开关、显示窗、脚垫、插口与标签必须分别成层，不能和机身烘焙成一片。
4. **遮挡和接触阴影**：旋钮底部、护板边缘、琴桥下方、面板缝和插口内侧需要小而明确的暗部；阴影只解释接触，不做脏污。
5. **功能标识**：型号、控制名、刻度、指示灯和箭头服务于“如何使用”，并按重要性排版。微型文字只能营造工艺，不能承担唯一操作提示。
6. **状态反馈**：让 LED、指针、开关位置、显示窗或局部高光改变；不要让整个物件发光、震动或换色。

## 3. 光线、材质与表面

- 全游戏实体物件统一使用**左上主光、右下落影**。一个物件最多 `1` 个主投影、`1–2` 个接触阴影。
- 金属不是镜子：避免大面积白色条纹、彩虹反射、过度复杂的多层渐变。金属只需要边缘亮线、局部冷暖变化和正确的凹凸层级。
- 喷漆 / 颗粒表面使用低对比 pattern，小点只落在面漆层；颗粒不能覆盖标签、螺丝和文字。
- 橡胶、塑料、布与木材比金属更哑，亮部更宽、更柔；不要把所有材质都画成镀铬。
- 做旧只标记受力边缘、手常接触处或螺丝附近；每个物件最多 `2–4` 处，不能用随机噪点铺满。

## 4. 可复用的机械词汇

建立可复用的 `Primitive`，但禁止复用整个物件模具：

- **紧固件**：螺丝、铆钉、螺帽；有中心槽、外圈与小阴影。
- **旋钮 / 旋轮**：外缘齿纹、指针、底部阴影、可读的刻度关系。
- **开关**：金属或塑料帽、根部、当前位置；拨杆和脚踏不应漂浮在面板上。
- **插口 / 槽**：外壳开孔、内圈、暗孔；只画正面可见部分，后方连线不能穿到前面。
- **显示与指示**：LED 有关闭颜色、激活颜色与小高光；刻度盘有玻璃 / 面板 / 指针三个层次。
- **边缘结构**：倒角、包边、面板缝、脚垫、接缝；它们是辨识“机械物”的主要来源。

复用 `Knob`、`Screw`、`Switch`、`Jewel`、`Jack` 等细节组件时，必须改变数量、大小、位置及其和外壳的关系，避免贴图式重复。

## 5. 小尺寸优先的层级

游戏中常见的 `64–96 px` 物件缩略图，玩家首先应读到：

1. 轮廓和厚度。
2. 操作件数量与位置。
3. 主材料与主色。
4. 激活状态。

只有在 `240–320 px` 的检视图中，才要求读到螺丝、刻度、型号、插口和细微做旧。缩小后若细节变成灰雾，删掉它，而不是继续加细节。

## 6. 三轮评审方法

1. **结构轮**：临时改为灰色平面；检查轮廓、厚度、装配缝、遮挡和接触点。此时仍能辨认对象才通过。
2. **材料轮**：加入材料、光照与硬件；检查每个高光是否来自同一方向，金属是否简洁，边缘是否比面部更有信息。
3. **缩小轮**：同时看独立大图和游戏内真实尺寸；大图检查工艺，缩略图检查识别与状态。SVG 必须渲染成 PNG 人工观察，不能只凭源码或构建成功判断。

## 7. 反模式

- 同一圆角矩形 + 换色 + 同一双旋钮，不算不同物件。
- 用厚黑描边代替真实厚度、倒角和材料交界。
- 为了写实把背面连杆、内部电路、隐藏金属件画到正面。
- 金属渐变过多、过死板，或所有材料都同样发亮。
- 无来源的随机划痕、噪点和镜面反射覆盖整面。
- 把真实品牌 Logo、型号、完整面板排版直接复刻进游戏资产。
- 用 Emoji、泛光图标、纯 CSS 卡片代替实体控制件。

## 8. 当前验证案例与交付要求

`EffectPedal.tsx` 的四块单块效果器是此基准的第一个验证案例；其特定轮廓与控制设计见 [effect-pedal-art.md](effect-pedal-art.md)。

以后新增任何拟物小物件，提交前必须同时交付：

- 原型说明（Object Card 的七项）。
- SVG 源码与真实尺寸截图。
- 至少一次结构 / 材质 / 缩小三轮检查记录。
- 对应的功能状态（若物件可操作）与激活态差异。
