# Tone Foundry — 乐器内容验证台 Visual QA

## 当前结论

Blackguard Standard 已按官方 1951 正视服务图、官方硬件规格与 1:1 五孔护板模板完成经典板式单切比例校准，并作为模块化金标准。其后七把母版补齐雕面单切、偏移、轮廓双切、中心块半空心、薄双角、无畏型钢弦与古典尼龙弦文化。当前装配台已拆成 `25.5 in bolt-on` 与 `24.75 in set-neck` 两套真实接口，共十二个实时文化预设；八把母版和双平台均已完成生产 SVG、页面交互与响应式复验。当前底座可继续扩展模块库，但仍明确定位为高质量拟物 SVG，而不是摄影级渲染。

## 已发现并修复

### P1 — 琴头与指板不能作为模块化基准

- **观察**：首轮 `350%` 分段检查发现琴头近似直板，弦钮轴套部分悬在木料外，旋钮过长且被错误装饰线串联；导弦器没有形成真实弦路。指板横向渐变分成三条色带，琴枕约两倍厚，品丝与弦阴影在放大后形成白色梯子和黑色竖栅。
- **结构修复**：重建非品牌化六联琴头轮廓，增加低音侧缓弧与端部配重；把六个轴套收回木料内并缩短旋钮，删除串联线；两根高音弦按“琴枕 → 按钮式导弦器 → 弦钮轴心”连续布线。琴枕降为 `4.2` 非缩放单位，品丝、边缘转面和琴弦阴影分别减细。
- **材质修复**：压平琴颈与指板的横向渐变和木纹对比，使其读成一整块枫木而非多层贴皮。
- **装配修复**：发现黑色护板绘制在指板之后，导致琴颈根部出现粗黑断层；将护板移到琴颈下方，拾音器与琴弦仍保持在上层。
- **复验**：按完全相同的四个 `350%` 机位连续完成 r0–r4 五轮截图；这些轮次解决了指板、琴枕、品丝和护板叠层，但随后证明琴头校验基准仍然错误，不能作为最终证据。

### P1 — 右手琴正面琴头被错误镜像

- **观察**：r0–r4 把六个弦钮旋钮画在画面右侧，并将轴心排成竖直一列；官方右手琴正面照中旋钮全部向画面左侧伸出，六个轴心从琴枕附近向右上斜排。旧轮廓也被主动泛化，无法充当经典款模块标准。
- **根因**：混淆了服务资料中的正面／背面关系，并错误地把“去品牌化”扩大成“改变类别轮廓”；后续数轮只在错误朝向内精修，没有重新验证基准方向。
- **修复**：以用户提供的正面产品照和 Fender American Vintage II 1951 右手琴官方资料重新建立琴枕坐标；六个轴心改为 `[282,184] → [319,49]` 的右上斜排，旋钮统一向左；琴头重建为左侧渐进展开、右侧端部圆鼓后连续收束的经典正面轮廓；导弦器移至琴枕上方并缩小，弦路重新连接到真实轴心。
- **复验**：r5 首先纠正镜像，r6 发现右侧曲线过度描成 S 形，r7 改回参考照的浅缓弧。`neck-r7-headstock-350.png` 与 `neck-r7-whole-100.png` 同时确认高倍结构和整琴第一眼方向；控制台无错误。

### P1 — 官方线稿不能替代实物 neck module 校准

- **观察**：用户补充的两张实拍显示，r7 琴头仍过宽且低音侧展开过度；弦钮轴套过小，导弦器类型错误。指板的琴枕过厚、heel 平切、品位点带金边且十二品双点过窄，均与实物不符。
- **修复**：以实拍琴头近景收窄轮廓并缩短低音侧展开；弦钮改为较大的封闭式轴套、金属中心和左向旋钮；圆形导弦器改为带螺丝的矩形压弦片。琴枕主线降为 `2.4` 非缩放单位；指板 heel 使用 `10` units 圆角；品位点改为无描边 `3.8` units 哑光黑圆点，十二品双点中心距扩大至 `19` units；枫木横向明暗进一步压平。
- **复验**：r8 完成结构重建，r9 消除弦钮“眼睛感”和过大的品位点。`neck-r9-headstock-350.png`、`neck-r9-mid-board-350.png`、`neck-r9-heel-pocket-350.png` 使用相同倍率与机位，`neck-r9-whole-100.png` 检查整琴第一眼比例。

### P1 — 指板仍读作平直图表，金属反射过于死板

- **观察**：r9 指板两侧转面、品丝和琴弦形成规则网格，heel 收口仍缺少实拍中的平底大圆角。桥板与控制板使用多段竖向 chrome 渐变和大量平行高光，放大后像程序化金属条纹。
- **修复**：指板 heel 改为平直底边与 `12` units 两侧圆角；枫木降饱和并加入两条低对比纵向细纹；品丝恢复可读的暗侧、金属面和单条低透明高光；高音弦从近白降为暖灰。大面积金属另建斜向宽反射，仅保留一条不对称弧形局部高光；桥板删去成对竖线，旋钮滚花由 `12` 条减至 `8` 条并降低对比。
- **复验**：r10 首次检查发现 heel 被误做成整段下弧，r11 按实拍改回平底圆角。`neck-r11-mid-board-350.png`、`neck-r11-heel-pocket-350.png` 与 `neck-r11-bridge-controls-350.png` 为同倍率复验证据。

### P1 — 将“指板旁边”误解为指板表面

- **观察**：用户用红圈标出的实际问题是指板右侧的 cutaway 与 upper horn。旧版内切浅而平，horn 偏尖；黑护板又贴近琴体边缘，导致轮廓在暗背景上消失。
- **修复**：以圈选实拍重建该段琴体 Bézier：U 形底部落在 heel 附近，向上快速展开；upper horn 改为短而饱满的圆角 lobe。护板内弧独立重画，并在内切和外缘保留连续的琴体退距；右侧厚度带从 horn 尖端移开，避免生成第二个角。
- **复验**：r12 首次截图出现一次性黑层，r12b 同状态未复现；r13–r14 检查发现整琴下 horn 仍读成细钩，r15 通过移动内弧而非继续拉外缘恢复横向体量。最终以 `neck-r15-heel-pocket-350.png` 检查局部，以 `neck-r15-whole-100.png` 检查整琴第一眼。

### P1 — 琴头金属件只有外观，没有真实装配结构

- **观察**：r15 的弦钮由同心圆和直接相连的横向胶囊组成；缺少六角锁紧螺母、独立弦柱、绕弦、侧轴和折面旋钮。压弦片也只是带螺丝的平面方块。
- **修复**：每枚封闭式弦钮拆为正面垫圈、六角螺母、凸起弦柱、两圈绕弦与短线尾，以及位于琴头背面的梯形折面旋钮。压弦片增加上折边、下暗边、两个走弦位置和中央螺丝。
- **二次纠错**：r16–r17 仍把传动短轴画在正面，并让旋钮与正面弦柱建立可见连接；这不符合实拍的前后装配关系。r18 将六枚旋钮及其背部结构整体移到琴头木板之前绘制、由随后绘制的木板遮挡重叠区域，删除正面短轴，并把旋钮端部放大约 `20%–25%`。正面现在只显示垫圈、六角螺母、弦柱和绕弦。
- **造型纠错**：r18 的旋钮仍是圆润胶囊，与实拍的短粗镀铬桨片不符。r19 先建立折面轮廓，但收腰过深、读作扳手；r20 改成连续外扩后又过于圆软；r21 最终采用窄根、连续外扩、平直上下边、削角外端的冲压件轮廓，并将高度提高到略大于正面轴套。材质只保留一块宽上反射与较暗的下折边，避免程序化条纹。
- **复验**：`neck-r21-headstock-350.png` 确认六枚旋钮具有与实拍一致的块状体量和折面结构，并且只从琴头外缘后方露出；`neck-r21-whole-100.png` 确认整琴尺度下没有过重。

### P1 — 局部精修掩盖了全身琴体与指板比例漂移

- **观察**：将 r23 全身图与正面产品参考按“琴头顶—琴身底”归一到同一高度后，琴身下半部偏窄、左上肩收束过早，右 horn 横向外伸且 cutaway 过深；指板从约 `44` units 的琴枕宽度扩张到约 `61` units 的 heel，锥度明显大于参考。旧琴身厚度边线也未随轮廓更新，在右侧形成游离黑线。
- **琴身修复**：重建上下肩、左右腰、下半部和 cutaway 的连续 Bézier；下半部先扩宽后按等高图回收至约 `348` units，使最终宽高比与参考接近；右 horn 收窄约 `8` units，cutaway 以 r24–r28 四轮在“过深—过浅”之间收敛。同步重画单侧厚度带并删除旧游离边线。
- **指板修复**：琴枕面宽从约 `48` 收至 `44` units，heel 从约 `61` 收至 `54` units；琴颈底层 heel 收至 `57` units。21 根品丝的横向展开改为每品约 `0.2` units，琴颈拾音器同步从 `82×29` 收至 `72×27` units，避免窄指板配超宽拾音器。
- **护板与表面修复**：护板左缘向内收，恢复琴身金色退距与桥板缺口；左肩均匀白色填充改为低透明渐变，避免高光改变轮廓判断。
- **复验**：r24 只检查新琴身，r25 加入新指板，r26–r27 修复护板和旧厚度线，r28 完成最终小幅比例收敛。`neck-r28-whole-100.png` 检查全身，`neck-r28-mid-board-350.png` 与 `neck-r28-heel-pocket-350.png` 检查指板锥度和 cutaway。

### P2 — 全身参考中的背带扣缺失

- **观察**：产品正面参考在左上肩和琴身底部各有一枚背带扣；r28 两处均为空，削弱了经典 slab 琴的完整装配感。
- **修复**：在琴身面板之前增加两枚背层镀铬背带扣，由琴身和侧板遮住安装根部，只露出真实正视图可见的冠部。
- **复验**：`neck-r29-whole-100.png` 确认上肩扣不改变肩线，底部扣不穿出地面阴影。

### P2 — 单体 SVG 缺少可复用几何契约

- **观察**：琴身路径、琴枕、弦鞍、穿弦孔、弦距和弦钮坐标散落在 `WorkshopGuitarSvg` 顶部，后续模块替换容易出现各部件独立漂移。
- **修复**：新增 `goldStandardGeometry.ts` 统一几何锚点与琴身正面/侧面路径；将琴颈木料、背部旋钮、琴头、指板、21 根品丝和品位点拆为独立 `GoldStandardMapleNeckSvg` 模块。
- **复验**：r30 验证几何契约抽取，r31 验证琴颈组件拆分；两轮的全身、琴头和中段指板 PNG 均与 r29 逐字节一致。

### P2 — 琴桥与控制组件仍与整琴实现耦合

- **观察**：桥板、桥拾音器、三枚弦鞍、控制板、拨档和旋钮虽已有语义分组，但仍内嵌于 `WorkshopGuitarSvg`，无法作为后续硬件组合和材质变体的稳定模块。
- **修复**：新增 `GoldStandardBridgeControlsSvg`，由统一几何契约注入桥位并保留各部件的 `data-guitar-part` 标识；父级继续后装琴弦，避免改变琴弦覆盖拾音器和弦鞍的真实层级。
- **复验**：r32 的全身、完整硬件和桥/控制板三张 PNG 与 r31 对应截图逐字节一致，证明拆分没有产生像素级视觉回归。

### P1 — Carved Crown 仍是概念图标而非可检视乐器

- **观察**：旧版使用近似双圆葫芦轮廓、皇冠状琴头和通用直颈；桥尾顺序、四旋钮位置、22 品指板、3+3 弦路和单切内弯均不准确，无法承受 350% 放大。
- **修复**：根据官方 1959 Reissue 正面图和规格建立 `carvedCrownGeometry.ts`，重画窄上肩、深腰、宽下腹与单切内弯；有效弦长固定为约 695 SVG units，对应 `24.75 in`。新增 22 品包边玫瑰木指板、正确由近到远分配的 3+3 弦钮、双封闭线圈、ABR 类桥、止弦尾件、三档拨片和四枚 top-hat 旋钮。
- **复验**：r1 暴露弦路交叉和夸张琴头；r2 修正弦钮顺序、护板和桥尾厚度；r3 收敛开书式琴头与单切轮廓。`crown-r3-whole-100.png`、`crown-r3-headstock-350.png`、`crown-r3-body-upper-350.png`、`crown-r3-hardware-350.png` 为同机位证据。

### P2 — Carved Crown 精细结构缺少模块边界

- **观察**：r3 的视觉结构已成立，但琴颈、琴头、硬件仍全部内嵌在整琴组件中，不利于后续更换指板、拾音器或桥尾组合。
- **修复**：拆出 `CarvedCrownNeckSvg` 和 `CarvedCrownHardwareSvg`，父级只保留雕面、漆层、琴弦、交互和装配顺序；模块使用 `data-guitar-module` / `data-guitar-part` 标识。
- **复验**：r4 的整琴、琴头与硬件 PNG 与 r3 对应截图逐字节一致。

### P1 — 三把琴的音色差异只能靠主观试听判断

- **观察**：实时 Web Audio 能正常播放，但原来的采样生成内嵌在 `ToneEngine`，无法固定种子离线测量；“三把琴不同”只有参数表，没有生产波形证据。
- **修复**：抽出 `pluckModel.ts` 作为浏览器与 QA 共用的纯采样模型；新增 `scripts/analyze-tones.ts`，固定 48 kHz、低 E 弦、力度 `0.82` 和 voice `0`，分析时长、起音、早/晚期 RMS 和 25–250 ms 差分能量。
- **复验**：Blackguard / Crown / Offset 时长分别为 `2.532 / 3.960 / 3.246 s`，明亮度代理值为 `0.56507 / 0.47986 / 0.52593`，Crown 起音 RMS `0.34393`，低于 Blackguard 的 `0.38744`。脚本对预期顺序设置硬断言。

### P1 — Offset Current 仍使用通用色块结构

- **观察**：旧版琴体像三个相交圆形，六联琴头、21 品指板、宽单线圈、双电路控制、螺纹弦鞍桥和长弦尾颤音均被符号化；整琴无法与官方 1966 正面结构对应。
- **修复**：新增 `offsetCurrentGeometry.ts` 与 `OffsetCurrentGuitarSvg.tsx`，建立 `25.5 in` 弦长、21 品包边深色指板、六联弦路、宽单线圈、rhythm / lead 控制区、螺纹弦鞍浮动桥、颤音板和摇把。`GuitarSvg` 不再保留旧通用绘制分支。
- **复验**：r1 的 100% 与五个 350% 机位确认完整结构，但暴露大琴头双鼓包和护板面积偏重；r2–r3 收回琴头为单一端部配重并修顺下缘。r3 又发现斜排轴心令四枚背装旋钮被木板遮没，r4 改为固定左外缘后，`offset-r4-headstock-350.png` 可见六枚旋钮都只从木板背后露出。偏移轮廓、护板面积与金属材质仍未结束。

### P2 — Offset Current 结构已成立但仍是单体组件

- **观察**：r4 的 21 品琴颈、六联琴头、双电路和浮动颤音仍全部内嵌于父级，不能独立替换指板、拾音器、控制电路或尾件。
- **修复**：新增 `OffsetCurrentNeckSvg` 和 `OffsetCurrentHardwareSvg`；父级只保留琴体厚度、漆层、琴弦、输入命中与装配顺序。模块和关键子部件均加入语义 `data-guitar-module` / `data-guitar-part`。
- **复验**：r5 的整琴、琴头、硬件与颤音四张 PNG 与 r4 对应图逐字节一致。

### P1 — Offset 琴体仍是居中圆泡，颤音板缺少装配结构

- **观察**：将官方正面图旋转为竖直方向并按琴头顶—琴身底归一后，旧路径的上下肩同时抬起、下腹居中，护板像中央圆片；桥和颤音尾件则是单层白色矩形/梯形，350% 下没有安装逻辑。

- **复验**：`offset-r6-whole-100.png` 检查新轮廓，r7 修圆左上角并以 `offset-r7-body-upper-350.png`、`offset-r7-vibrato-350.png` 检查护板与金属总成。护板和旋钮仍需最终横向比较。

### P1 — 深色护板吞掉上 horn，rhythm 电路被画成黑色面板

- **观察**：r8–r9 虽已连续化上 horn 曲线，但暖棕护板在暗背景里读成琴体缺口，使上角仍像细尖；rhythm 电路由 `58×88` 黑色矩形承载，与官方护板内嵌拨档/滚轮结构不符。
- **修复**：改用官方海绿色正面参考的 aged parchment 材质逻辑，并在视觉文档中同步色彩规则；删除大黑面板，改为一枚小拨档和两枚带刻纹的滚轮槽。两枚主旋钮增加接触阴影、裙边圈和位置指示线。
- **复验**：`offset-r10-whole-100.png` 首次确认浅护板恢复琴体读取，`offset-r11-whole-100.png` 与 `offset-r11-body-upper-350.png` 确认双电路控制不再遮断 horn，且宽单线圈、浮动桥和颤音总成层级清晰。

### P2 — 控制旋钮与拨档仍像图标化零件

- **观察**：r21 的两枚控制旋钮使用深色径向渐变，放大后像黑色玻璃球；拨档由两条平行斜线组成，没有安装槽、金属拨片和开关帽。桥板边缘也缺少冲压件的翻边关系。
- **修复**：控制旋钮拆成接触阴影、镀铬外圈、低对比滚花侧圈和独立金属顶面；拨档增加暗色行程槽、窄金属拨片与黑色削角帽。桥板增加上缘高光、左右翻边和下缘折面；三枚黄铜弦鞍增加连续圆柱高光。
- **复验**：`neck-r22-hardware-full-350.png` 首次确认部件结构，`neck-r23-hardware-full-350.png` 在漆面修复后再次以相同机位确认，没有新增高光冲突或遮挡错误。

### P2 — 琴身下半部反射读作平直色块

- **观察**：r22 的琴体下部使用一块接近水平的深色半透明路径，350% 下像覆盖层而不是硝基漆中的环境反射；上肩高光也缺少面积衰减。
- **修复**：上下两块反射分别使用独立的受控渐变。下部反射沿下腰曲率向右上抬升，并从 `0.02` 逐步过渡到 `0.15` 透明度；上肩高光沿左上主光方向衰减到透明，不再使用均匀白色填充。
- **复验**：`neck-r23-hardware-full-350.png` 检查局部曲率，`neck-r23-whole-100.png` 检查整琴第一眼；下半部不再出现水平色带。

### P1 — 吉他下缘被检验台裁切

- **观察**：SVG 按宽度计算后高度超过检验垫，琴体下缘被裁掉。
- **影响**：收藏物不完整，破坏第一印象与直接拨弦区域。
- **修复**：SVG 严格保持 `1:2`，为工具条与提示区保留安全区；桌面宽 `319 px`、移动端宽 `231 px`，高度由 viewBox 比例计算。
- **复验**：360×740 为 `231×462 px`、390×844 为 `231×462 px`、1024×768 为 `319×638 px`，均显示完整琴头与琴尾。

### P1 — 检视手势与拨弦争抢输入

- **观察**：若单指既拖动琴体又拨弦，玩家无法预判一次触摸会发声还是移动画面。
- **影响**：放大查看与演奏都不可靠，移动端尤其容易误触。
- **修复**：显式分为“检视 / 演奏”两态；检视态处理拖动、双指、滚轮和双击，演奏态把单指交给琴弦。
- **复验**：100%–350% 缩放、205% 拖动、复位、演奏模式切换均可恢复；倍率常驻可见。

### P2 — 放大后品丝与琴弦缺少离面层次

- **观察**：整琴尺寸下可接受，但 200% 后品丝和弦像直接画在木材上。
- **修复**：每根品丝增加暗侧、亮面与细高光；六根琴弦增加按粗细变化的接触阴影；补充琴颈袋阴影和琴体连续反射线。
- **复验**：205% 近景可区分弦、品丝、指板和拾音器的前后关系。

### P1 — 琴颈出现粗外轮廓

- **观察**：琴颈外框 stroke、纵向深色渐变、过宽的琴颈底层和琴颈袋阴影叠加，放大后读作一圈粗棕黑描边。
- **修复**：取消琴颈外框式 stroke；木纹渐变改为横向转面；琴颈底层从每侧露出 `16–17` 个单位收至约 `4` 个单位；接缝降至 `0.6` 非缩放线；琴颈袋阴影收至每侧约 `3` 个单位。
- **复验**：`310%` 下琴颈两侧读作窄木材转面，不再形成独立黑框。

### P1 — 原创变体不能充当经典比例基准

- **观察**：早期 Workshop Slab 为了原创识别度夸张了上肩、cutaway 和护板，无法判断后续模块是“风格差异”还是“基础比例错误”。
- **修复**：将 F01 重建为 Blackguard Standard：按公共领域板式单切轮廓校准琴身宽高和圆头 cutaway；按 `25.5 in / 648 mm` 弦长公式计算 21 根品丝；使用 bolt-on 枫木琴颈、双单线圈、单层黑护板、三鞍穿体桥和两旋钮控制板。
- **复验**：100% 可读出经典板式单切家族；205% 可数出 21 品并辨识三枚黄铜弦鞍。

### P1 — 经典硬件方向与尺寸不准确

- **观察**：上一稿把琴颈拾音器画成纵向长条，琴颈与桥端弦距过窄，控制板长度仅约标准件的七成，导致琴体即使轮廓接近也不像真实经典结构。
- **资料校验**：Fender 官方 American Vintage II 1951 产品规格确认 `25.5 in` 弦长、`1.650 in` 琴枕、21 品、双单线圈与三枚黄铜桶形弦鞍；官方服务图确认横向封闭式琴颈拾音器、斜置桥拾音器及硬件相对位置；Fender 控制板规格为 `6.27 × 1.26 in`；vintage 桥板资料为 `3.93 × 3.148 in`。
- **修复**：以约 711 SVG units 表示琴枕至弦鞍接触点的 25.5 in 有效弦长；琴枕扩至约 46–48 units，琴颈袋控制在约 63–65 units，桥端外弦距扩至约 60 units；琴颈拾音器改为横向无裸露磁柱的镀铬胶囊，桥拾音器横跨六弦并带约 8 度斜角；控制板重建为约 `174 × 35` units。
- **复验**：100% 下两枚拾音器、桥板、三弦鞍和控制板的方向可辨；205% 下六弦从琴枕向桥端连续展开，硬件不再挤成中轴细条。

### P1 — 琴身曲率缺少可追溯校准

- **观察**：连续手调造成上肩、腰线、下半部和 cutaway 互相补偿，局部改善后整体比例仍漂移。
- **修复**：从 Fender 官方服务图正视页提取琴体外轮廓锚点，以琴颈中线归一化后重建原创 Bézier 曲线；使用 Warmoth 1:1 五孔护板模板校准护板的颈袋开口、拾音器开口、桥板缺口与五枚安装螺丝关系。
- **边界**：校准层只用于结构比例；不保留品牌 logo、签名、官方零件号或可识别琴头字样。
- **复验**：第二轮 100%、205% 与 310% 截图均从同一运行页面生成，后续修改必须继续使用相同状态复验。

### P1 — 左上肩误用了内部线稿锚点

- **观察**：第一次读取官方服务图时，将图中的内部引线／结构线混入琴身外轮廓，左上肩因此收成近圆形小包，腰线位置也被提前。
- **修复**：按端点连通关系重新分组官方矢量线稿，得到由 52 个连续段组成的单一琴身外轮廓链；用该链重新归一化上肩、左右腰、下半部、horn 与 cutaway。
- **复验**：100% 下上肩与下半部宽度关系恢复；205% 下左侧轮廓从上肩、腰线到下半部连续，无突然内缩。

### P2 — 木纹方向与漆面反射偏插画化

- **观察**：旧木纹横向横跨琴体，并叠加数条粗弧形高光，放大后像装饰线而不是顺纹白蜡木与透明漆。
- **修复**：改为沿琴体长轴延伸的纵向 ash grain，暗纹控制在 `1.6–1.9` units、低于 `0.20` opacity；粗描边式高光替换为两块有面积和衰减的透明反射；侧板由近黑调整为受阴影的暖棕。
- **复验**：205% 下纹理随琴体长轴连续，右侧厚度不再读作独立黑色描边。

### P1 — 三个家族轮廓过于接近

- **观察**：初版主要依赖颜色、护板和拾音器区分，琴体均接近圆润葫芦形。
- **影响**：无法承载文化家族，盲盒内容会退化成换色。
- **修复**：Blackguard Standard 独立重画琴体、单切凹口、护板负形、琴颈、琴头和硬件比例；其余两把保留作旧版对照。
- **复验**：Blackguard Standard 已形成经典板式单切参考结构，但拟物精度仍需继续确认。

### P1 — 材质过于扁平

- **观察**：琴体、护板和金属最初只有单层渐变。
- **影响**：读作高级图标而非收藏级乐器。
- **修复**：增加琴体厚度、透明漆层、矢量木纹、边缘倒角、高光带、接触阴影、三层护板断面、独立桥鞍、螺丝、滚花旋钮、弦钮轴套与完整弦路。
- **复验**：放大桌面视图能读出硬件与表面层级。

### P1 — 尖角处琴身厚度分裂

- **观察**：旧实现将完整琴体路径平移 `11×14` 后作为厚度层；圆弧处可以伪装成侧板，但 cutaway 与 horn 会出现第二个尖角，读作两片琴身。
- **修复**：删除整琴复制层；厚度改为仅覆盖右侧和下缘的专用 `BODY_SIDE` 带。正面负责接缝，侧板另画一条后缘线，尖角不再生成厚度。
- **复验**：在 `205%` 与 `310%` 下，右侧至下缘为连续单侧板；horn 只有正面轮廓，没有第二片偏移轮廓。

### P2 — 放大后勾边过粗

- **观察**：SVG 随 CSS 放大时，琴身、琴颈、护板和硬件 stroke 同比增长，`205%–310%` 下出现插画式黑边；护板米白断面也过宽。
- **修复**：主体结构边改为 `vector-effect="non-scaling-stroke"`；琴体正面降至 `1.5`、侧板后缘 `1.4`；琴颈、护板与硬件边界同步减细。护板内外路径间距收窄约一半。
- **复验**：`310%` 下外轮廓保持细线，护板仍能读出分层但不再像粗描边。

### P2 — 琴身比例偏葫芦形

- **观察**：上肩过大、腰线浅、右侧 horn 外伸过多，整琴更像符号化葫芦轮廓。
- **修复**：收小上肩并加深左腰；右侧 cutaway 改为连续曲率，horn 仅略宽于琴颈；护板从圆盘状收成窄流线结构。
- **复验**：100% 下家族轮廓更紧凑；310% 下 horn 不再由侧板生成独立木刺。

### P1 — 不稳定 SVG 材质滤镜

- **观察**：`feTurbulence` 木材纤维在 headless Chromium full-page 合成中产生黑层。
- **影响**：存在 Aigram WebView 合成风险。
- **修复**：撤除程序化噪声滤镜，并进一步删除投影使用的 `feGaussianBlur`；琴底软影改为两层径向渐变椭圆。拖动层由永久 `translate3d + will-change` 改为 2D transform，并由检验台提供 paint containment。
- **复验**：100%、无拖动 205% 与 310% 的 GPU 截图正常；headless Chromium 的拖动后截图在 GPU 模式仍偶发黑色合成层，但以 `--disable-gpu` 重拍同一状态完整正常，页面控制台无错误，判定为测试环境 GPU 合成问题而非 DOM/SVG 内容黑层。

### P2 — Clean / Drive 单侧触控不足 44 px

- **修复**：单侧按钮提高到 44 px，组合开关提高到 52 px。
- **复验**：自动检查无短触控按钮。

### P1 — 内容底座只有电吉他，无法代表声学吉他文化

- **观察**：Blackguard、Crown 与 Offset 已覆盖三种电吉他结构，但它们都以实心琴体、电磁拾音器和金属琴桥为核心。即使轮廓不同，评审页仍不能回答空心箱体、音孔、木桥和空气共鸣如何进入模块系统。
- **结构修复**：新增 `25.4 in`、20 品、十四品接柄的 Timber Dreadnought；独立建立宽下腹箱体、云杉面板、桃花心木侧板、实心方锥 3+3 琴头、乌木指板、音孔、多圈 rosette、龟纹护板、belly 木桥、补偿弦枕和六枚弦钉。首轮琴头过圆、护板过大，第二轮分别改为方锥轮廓和贴音孔展开的非对称泪滴。
- **声音修复**：除独立 tone fingerprint 外，在生产采样模型加入约 `102 Hz` 空气腔模态与 `190 Hz` 面板模态；Clean 滤波链的箱体峰值约 `108 Hz`、短空间延迟 `42 ms`。固定种子测得时长 `3.552 s`、起音 RMS `0.36263`、明亮度代理 `0.51127`，均形成不同于三把电吉他的签名并通过顺序断言。
- **复验**：`desktop-1024x768-acoustic.png` 检查全身与四琴选择器，`mobile-360x740-acoustic.png`、`mobile-390x844-acoustic.png`、`mobile-430x932-acoustic.png` 检查移动端；`review-desktop-1440x1000.png` 与 `review-mobile-390x844.png` 检查四琴等尺度、木吉他实时局部裁切和音色表。
