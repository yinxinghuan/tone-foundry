# Tone Foundry — Sonic Index Visual Bible

## 1. Visual thesis

- **Audience**：以英文为主语言的手机用户；喜欢吉他文化、收藏、声音探索与轻量创作，不要求掌握专业音频软件。
- **Emotional promise**：玩家不是在填写工单，而是在建立一份可以听见、可以收藏的乐器标本。
- **Thesis**：真实而精细的模块化吉他，被放进一套当代乐器目录、工业设计索引和声音标本馆组成的明亮编辑系统中。
- **Signature moment**：玩家点选部件时，索引线从名称连接到吉他对应位置，部件即时显影，声音指标同步移动。
- **Required qualities**：清醒、精密、可收藏。

## 2. Composition and camera

- 主要目标为 `360–430 px` 手机竖屏，兼容桌面。
- 吉他保持正视产品角度和统一 `600 × 1200` SVG 坐标系。
- 制琴页使用上方标本舞台、下方单问题选择区；吉他始终是第一焦点。
- 入口用“未装配”吉他的工程轮廓、`05 MODULES` 标签和单个主动作回答“开始一份新标本吗”；制琴只回答当前部件选择；成琴档案只回答“它是什么、听起来如何、下一步做什么”。
- 手机单屏不超过一个主标题、一行辅助说明、2–3 个候选与一个主动作。
- 顶部和底部遵守安全区；工具不遮挡琴头、琴桥或候选按钮。
- 查看器支持 `80%–200%` 缩放；制琴自动部位推镜与手动缩放叠加。
- 调音是独立全屏状态：完整吉他占舞台，声音指标、效果器链和单一试听控制占下方 / 右侧阅读区；琴体、琴颈、拾音器、琴桥、饰面按组直接列出本局已开出的候选，点选后在同一画面试装并试听，不能缩进制作预览框或跳回制琴流程。
- 效果器以小型拟物单块 SVG 表达：每一块必须是不同硬件原型而非换色同模。Boost 使用斜面黄铜前级、单颗滚花大旋钮和 jewel LED；Overdrive 使用三旋钮橙色性能脚踏；Chorus 使用宽钢壳、紫色丝印面板、单大 Rate 盘和 Depth 拨杆；Tape Echo 使用颗粒深绿紧凑壳、三颗黑色滚花旋钮和 Mod 按钮。全部包含螺丝、侧面输入 / 输出口、金属脚踏、型号丝印和受控磨损。链路用等宽缩写与箭头表示；外围布局继续遵守 Sonic Index 的细线和网格，禁止改成暗色机架或泛滥的发光效果。

## 3. Color

- **Paper** `#F2F0EA`：页面与主表面。
- **Warm White** `#FAF9F5`：被选中的区域与阅读面。
- **Graphite** `#181917`：主文字、边框与主按钮。
- **Steel** `#A7AAA6`：结构轮廓、禁用态与辅助数据。
- **Signal Green** `#B8D63D`：选择、播放、进度与成功。
- **Rust** `#A74C32`：索引线、稀有部件和强调数据。
- **Error Red** `#B23B32`：错误与恢复。
- **Usage**：纸白 76%，真实吉他颜色 18%，功能色 6%。不使用渐变按钮、玻璃、霓虹或大面积深色背景。

## 4. Typography

- 英文主语言使用 `"Helvetica Neue", Arial, sans-serif`；中文使用 `"PingFang SC", "Microsoft YaHei", sans-serif`。
- 数字、编号、BPM 与技术索引使用 `"SFMono-Regular", Consolas, monospace`。
- 不使用宋体、楷体、手写体、窄体复古字体或装饰字体。
- 手机标题 `28–42 px`；问题标题 `24–30 px`；正文 `16 px`；候选名称 `14–16 px`；按钮 `13–15 px`。
- 可读索引 `11–12 px`；只有不需要阅读的纯辅助数字允许 `10 px`。任何操作、名称或状态不得低于 `11 px`。
- 英文技术标签可用 uppercase 与 `0.04–0.08em` 字距；中文不额外拉开。

## 5. Shape, material, and lighting

- UI 以直角、1 px 细线、基线网格、裁切标记、编号与索引线构成。
- 普通面板 `0–4 px` 圆角；只有头像、播放按钮和状态点可以使用圆形。
- 页面不模拟旧纸、收据、印章、皮革、琴盒、黄铜按钮或工作台。
- 真实材质全部集中在吉他 SVG：木纹、漆面、金属、塑料、弦与磨损继续服从左上主光。
- UI 阴影只用于悬浮的音色面板和必要的层级分离，模糊半径不超过 `18 px`。

## 6. Characters, environments, and assets

- 游戏没有角色，吉他是唯一主角。
- 继续使用现有八把校准母版、十一种可玩电吉他基础琴型、两个模块平台和语义化 SVG 分组。
- 未确认模块以冷灰工程轮廓表达，不能显示成一把已经完成的灰色实体琴。
- 已确认或临时试装模块保持真实材质；吉他细节质量、比例与模块锚点规则沿用校准系统。
- 两个平台各有 8 种饰面；金属颗粒、透明木纹、烟草外缘与轻度做旧必须是独立表面层，不能让所有饰面退化为换色渐变。扩展琴体的结构卡与渲染检查见 [body-family-art.md](body-family-art.md)。
- 公共墙缩略图必须在 `160 × 160` 仍看出琴体家族与一个结构性部件。

## 7. UI and icons

- 自绘 `24 × 24` 线性 SVG 图标，`1.6–1.75` stroke；禁止 Emoji。
- 所有触控目标至少 `44 × 44 px`。
- 主按钮为 Graphite 实心矩形；选中行为 Signal Green 底色 + 黑色单选标记，不只依靠颜色。
- 制琴候选在手机上使用至少 `52 px` 高的索引行；英文名称允许省略，但完整含义需在当前主吉他旁的试装标签中出现。
- 制作过程只保留一次即时试听与备用缩放控制；音色指纹和效果器链只在五部件完成后的调音页面出现。缩放控制不覆盖触摸画布，单指拖拽、双指缩放和双击复位在制作、调音与详情中都可用。
- Riff 使用四小节分页，每格至少 `44 px`；播放头同时使用边框与位置反馈。
- 琴架与公共墙使用编辑目录网格，不使用漂浮卡片；作者头像与名称保持可点击。
- 详情页由可缩放吉他、名称与作者、First Riff、五项模块清单和六维音色构成。

## 8. Motion and VFX

- 动效清醒、机械、短促；fast `140 ms`、standard `240 ms`、camera `560 ms`。
- 候选按下同一帧出现选择反馈，`≤80 ms` 更新吉他。
- 索引线在 `220 ms` 内伸向当前部件；指标条在 `240 ms` 内重排。
- 镜头聚焦仍使用 transform/opacity；不使用弹跳、粒子雨、屏幕摇晃或发光。
- 稀有完成态只允许 Signal Green / Rust 的短暂面积变化。
- `prefers-reduced-motion` 下取消位移与伸线，只保留最终状态。

## 9. References translated into principles

- **Swiss editorial systems**：明确基线、编号和留白；不照搬海报中的微型字体。
- **Industrial design catalogs**：让结构、名称与数据相互指向；不把产品变成冷漠参数表。
- **Museum specimen labels**：每把琴有身份、来源与可比较指标；不使用仿古博物馆装饰。
- **Contemporary music publishing**：英文标题具有节奏和文化感，但不牺牲移动端阅读。

## 10. Anti-patterns

- 禁止旧版收据、蓝墨、QC 章、手写圈、楷体、琴盒、皮革、木台和金属渐变按钮。
- 禁止 `6–10 px` 的玩家可读文字、用大字距补偿小字号、把所有英文做成微型 uppercase。
- 禁止通用 SaaS 卡片、玻璃拟态、大圆角胶囊、霓虹渐变与装饰性模糊。
- 禁止为了放大文字而把吉他压缩成缩略图；应先删除重复说明。
- 禁止选中候选后到下一步才更新吉他。
- 禁止在手机上常驻完整导航、步骤名称、平台规格或技术 manifest。

## 11. Vertical-slice acceptance

- 英文为设计主语言；无手动设置时按系统语言自动选择中文或英文，`localStorage.game_locale` 的手动设置优先。
- 入口、制琴、成琴档案、Riff、琴架、公共墙和详情页属于同一个 Sonic Index 系统。
- `360 px` 宽度下无横向溢出；所有按钮至少 `44 px`；操作文字至少 `11 px`。
- 制琴页第一眼看到吉他、当前问题、候选与主动作；无需滚动。
- 候选切换即时改变吉他，试听与音色指标读取当前临时组合。
- 档案页能清楚阅读名称、BPM、稀有分、部件与音色。
- 静态 UI 审计通过；真实运行截图覆盖入口、制琴、调音、完成、Riff、墙和详情，并修复全部 P0/P1。

## 12. Reusable pedal drawing pattern

- 所有拟物小型物件遵循 [skeuomorphic-object-svg.md](skeuomorphic-object-svg.md)：先建立原型、装配逻辑和材质层，再通过结构 / 材质 / 缩小三轮检查。单块效果器的专用轮廓与控制规则补充在 [effect-pedal-art.md](effect-pedal-art.md)；新增效果器必须先选独立硬件原型，禁止用同一个外壳换色扩展效果库。
