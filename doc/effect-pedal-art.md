# Tone Foundry — 拟物单块效果器 SVG 绘制模式

这份模式用于 Tone Foundry 后续新增单块效果器，以及其他需要“小型真实硬件”的 SVG 资产。目标不是画一个颜色不同的按钮，而是让玩家在很小的尺寸下仍能读出它是哪一种器材、如何控制、为何会改变声音。

## 1. 先选硬件原型，不能先选颜色

每一块效果器必须先指定一种**结构原型**，然后才选择表面与颜色。一个效果器至少要与另一个效果器在以下三项中有两项不同：

- 外壳轮廓：斜面、宽壳、紧凑长方形、带性能脚踏的大壳等。
- 控制架构：1 个大旋钮、3 个小旋钮、旋钮 + 拨杆、旋钮 + Mod 按钮等。
- 脚踏 / 面板语言：上翻性能板、裸露金属脚踏、银色面框、独立铭牌等。

当前四个基准：

| 效果 | 原型 | 关键识别点 |
| --- | --- | --- |
| Boost / Range Lift | 老式前级 | 斜面黄铜壳、单颗滚花大旋钮、jewel LED |
| Overdrive / Crank | 性能过载 | 三旋钮橙色壳、宽大脚踏板、中央状态灯 |
| Chorus / Widener | 宽钢壳合唱 | 紫色丝印面板、单大 Rate 盘、Depth 拨杆 |
| Tape Echo / Loop Line | 紧凑模拟延迟 | 深绿颗粒喷漆、三颗小旋钮、Mod 按钮 |

品牌、Logo、型号文案不能直接复制真实产品。可借鉴真实器材的硬件语法，但名称、丝印和构图必须是 Tone Foundry 自有资产。

## 2. 固定分层顺序

从下至上按以下顺序绘制，避免“贴图感”：

1. 投影：仅 `1` 个短而软的投影，方向统一为左上主光下的右下落影。
2. 外壳：使用 `path` / `rect` 做出独特轮廓，保留 `2–3 px` 深色边缘。
3. 漆面或金属：一层方向性线性渐变；颗粒漆只用低对比小点 pattern，不使用噪点滤镜覆盖所有区域。
4. 内面板 / 铭牌：用更深或更浅的独立面，形成可维修、可拆卸的机械结构。
5. 硬件：螺丝、旋钮、LED、脚踏、拨杆、侧面输入 / 输出口。
6. 丝印：型号 > 效果类别 > 控制名 > 次级技术字。越次级的字越小、对比越低。
7. 激活态：只改变 LED 颜色与 `3 px` 的轻微上浮 / 阴影，不能把整个踏板发光。

## 3. 可复用细节组件

`EffectPedal.tsx` 中的 `Knob`、`Screw`、`Switch`、`Jewel`、`Jack` 是统一的细节词汇。新增效果器优先复用它们，再改变数量、尺寸、位置和面板关系。

- `Knob`：要有齿纹、指针、高光与轻微阴影；旋钮不应是无标记圆形。
- `Screw`：至少两个可见螺丝，让面板具备装配逻辑。
- `Switch`：要有金属帽、中心暗孔、上方高光和落影。
- `Jewel`：关闭为暗红，激活为 Signal Green，并保留白色小高光。
- `Jack`：只作为侧面局部出现，不把后方连线或内部金属件画到正面。

## 4. 小尺寸的取舍

- 游戏内选项宽度约 `72–86 px`：优先保留轮廓、旋钮数量、脚踏和 LED；微型文案只起质感作用，不能承担操作信息。
- 评审图宽度约 `260–320 px`：完整保留型号、控制名、螺丝、插口和表面颗粒，以此检查比例与材质。
- 不用泛光、镜面大反射、玻璃感或过强金属渐变。它们会在缩小后变成脏灰块。
- 同一行四块踏板必须具有不同的外轮廓和负形；去色后也应能区分。

## 5. 必经检查轮次

每次新增 / 大改按三轮检查：

1. **轮廓检查**：把填充暂时改为灰色；四块放在一起时能否按轮廓识别？
2. **材质检查**：恢复表面后，确认投影、喷漆 / 金属、螺丝和脚踏都在同一左上主光下，且没有过度复杂的高光。
3. **缩小检查**：在游戏内实际选择尺寸与独立 1600 px 评审图各看一次；前者读结构，后者读工艺。

SVG 变更必须先渲染成 PNG，再人工观察，不能只根据源码或构建成功判断视觉质量。

## 6. 禁止项

- 禁止四块踏板使用同一圆角矩形、同一双旋钮、同一脚踏布局，只更换颜色。
- 禁止真实品牌 Logo、型号或可辨识的面板复刻。
- 禁止把电路、背部走线或被正面遮挡的金属连杆画在面板前方。
- 禁止用 Emoji、通用发光图标或纯 CSS 渐变按钮代替实体零件。
- 禁止为了“写实”而加入无法在 80 px 尺寸辨认的随机杂讯。
