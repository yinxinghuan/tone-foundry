# Tone Foundry / 音色铸造所 — Requirements

> 当前状态：完整开发模式。内容底座、五步限量抽取制琴、收藏存档、16 步 First Riff 与公共琴墙的大循环已进入代码；长按选品、四小节编曲、Remix 与排行筛选保留为后续深度迭代。

## 1. Overview

《Tone Foundry / 音色铸造所》是一款面向手机竖屏的制琴收藏与音乐创作游戏：玩家组合模块化吉他、获得由制琴参数决定且可复现的独特音色，在六弦步进音序器中完成作品，并将吉他与作品发布到公共琴墙供其他玩家试听和 Remix。

### 玩家与体验场景

- 目标玩家包括喜欢吉他、乐器收藏、音乐创作或精致物件，但不一定理解专业音频软件的普通用户。
- 单次首次体验目标为 6–10 分钟；回访试听、改曲或浏览琴墙目标为 2–5 分钟。
- 核心情绪承诺是“我亲手造出了一把只属于我的琴，而且别人能听见它的声音”。
- 主要交互依次是选件造琴、拨弦试听、在音序器中点格创作、发布与 Remix。

### 主要状态

1. 标本入口：以一把未装配的工程轮廓、五个模块提示和一个明确的开工动作，引导玩家开始新的制琴局；琴架与公共琴墙仅作为次级入口。
2. 制琴台：逐项选择琴体、主体材质、琴颈、拾音器、琴桥、琴弦和表面处理。
3. 成琴揭晓：展示完整吉他、音色指纹、收藏编号、命名与标准试听。
4. 试琴室：支持逐弦拨奏、扫弦、延音和 Clean / Drive 通道切换。
5. Riff Lab：六弦步进音序器、播放控制、速度、音阶、预设与录入工具。
6. 发布结果：保存吉他与 First Riff，生成公共琴墙条目。
7. 个人琴房：浏览、比较、重新试听和继续编辑自己的吉他。
8. 公共琴墙：浏览其他玩家的吉他，试听作品，打开作者信息并发起 Remix。
9. 加载、空、离线、生成失败、保存失败、发布成功与恢复状态。

### 玩家界面简报 / 极简开箱流程

- **当前玩家问题**：此刻只需要决定“要不要打开”或“这三个部件装在琴上哪个最好”，不需要理解完整制造流程、工单结构或后台工作区。
- **唯一焦点**：制琴页面始终以完整吉他为中心；未开箱时焦点是包裹，开箱后焦点是吉他上的即时试装结果。
- **未装配语义**：新工单不能以一把已经完成的实体琴作为默认预览。首次开箱前只显示由琴体、琴颈、硬件和弦路构成的灰银结构轮廓；玩家确认部件后，相应区域才获得实体材质。尚未确认的区域始终保留为轮廓。
- **部位镜头**：进入候选选择时，镜头在 `520–680 ms` 内从整琴平滑推近当前部位；琴体与饰面聚焦下半部，琴颈聚焦上半部，拾音器和琴桥聚焦琴身中央硬件。确认后镜头退回整琴，再进入下一个包裹。
- **导航原则**：制琴过程中不显示常驻工作区导航、页面标题、工单号码、平台规格或五个带文字的步骤。进度只允许使用 5 个直径 `4–6 px` 的静默圆点。
- **单屏信息预算**：除吉他外，最多同时显示 1 个短标题、1 行辅助说明、2–3 个候选和 1 个确认动作；任何技术编号均降为候选卡上的低对比辅助信息。
- **直接试装**：触摸候选后，同一帧进入选中态，并在 `≤80 ms` 内把该候选临时应用到主吉他预览；切换候选立即替换临时预览。只有点击确认后才写入本局正式配置并进入下一包。
- **显影规则**：临时试装除更新几何与材质外，还要让当前实体模块从灰色结构层中显现；切换候选时可逆，未来模块不能提前实体化。
- **制琴检视**：琴盒舞台保留精简的试听、放大、缩小和复位工具。候选被选中后可以播放 `2.4 秒` 的统一比较句；琴体、琴颈、拾音器、琴桥和饰面都必须对合成参数产生可辨但不过度的影响，不能用同一段母版音色冒充模块试听。
- **调音时机**：制琴中的主预览不显示完整音色面板；五个模块确认后必须进入一个独立、全屏的调音步骤。该步骤按琴体、琴颈、拾音器、琴桥、饰面同时列出本局已开出的候选；点选任一兼容候选会立即替换吉他、更新六维音色并播放统一 `2.4 秒` 比较句，不跳转页面也不要求二次确认。主按钮确认成琴后才进入制造档案。
- **直接检视**：制作舞台、调音舞台与公共琴墙详情均支持单指拖动平移、双指缩放和双击缩放 / 复位。缩放按钮只是可发现的备用控制，不能阻断或替代触摸手势；视口的触摸区域使用 `touch-action: none`，控制按钮保持在独立层。
- **平台信息**：弦长和接柄平台继续影响兼容池与 SVG，但玩家流程不常驻展示；仅在成琴制造档案中作为次级信息出现。
- **次级入口**：琴架和公共琴墙只在开始页底部或结果页出现，不在制琴中常驻。内部自由拼装和试音验证继续只存在于 `review.html`。

### 环境约束

- 主要平台为 Aigram 内嵌 WebView，优先手机竖屏；目标宽度覆盖 360–430 CSS px，并适配 768–1440 px 桌面浏览。
- 主要输入为触摸和 Pointer；桌面支持鼠标与键盘辅助操作。
- 所有交互目标至少 44×44 CSS px；移动端不依赖 hover。
- 支持中文与英文，所有长名称和按钮需承受英文文本约 1.6 倍的扩张。
- 尊重安全区、静音状态与减少动态效果设置；无法启动声音时提供清晰的重新启用入口。

## 2. Visual Design

正式界面采用 **Sonic Index / 声音标本馆**：真实高端吉他保留完整拟物材质，所有操作与信息则进入明亮、精密的当代乐器目录系统。英文是设计主语言；实际启动时根据系统语言自动显示中文或英文，玩家的手动语言选择优先于系统判断。

手机端的字号底线是体验规则而非装饰偏好：正文至少 `16 px`，候选名称 `14–16 px`，按钮 `13–15 px`，可读技术索引 `11–12 px`；只有不承担阅读的纯辅助数字允许 `10 px`。界面通过删除重复说明、缩短标签与重新分配区域解决拥挤，不能以压缩吉他或缩小文字换取空间。

吉他主体采用精细模块化 SVG 绘制，不以单张整体图片替代。内容校准架必须包含至少 8 把独立母版，用来覆盖不同弦长、接柄、琴身构造、桥尾和琴头语言：

- 4 把现有母版：五十年代板式单切、雕面单切、六十年代偏移、十四品无畏型木吉他。
- 4 把新增母版：25.5 英寸轮廓双切 SSS、24.75 英寸半空心双切、24.75 英寸轻薄双角实心、650 mm 十二品接柄尼龙弦古典吉他。
- 新增母版必须拥有独立正面轮廓、琴颈 / 琴头、桥尾与识别硬件，不能用现有琴体路径缩放、镜像或只换色完成。
- 八把母版在评审架上统一使用 `600 × 1200` 画布与相同显示尺度；缩略图至少能读出琴身家族和一个结构性识别件。
- 后四把的完成标准与前四把相同：全身视图必须包含可信的琴体厚度、边缘倒角 / 包边、漆面转折、独立琴头、完整品丝、弦钮正背层级、真实拾音器 / 音孔、桥尾、控制件、螺丝与连续六弦；不能保留圆形旋钮占位、胶囊状拾音器或单层渐变琴体。
- 每把新增母版必须分别通过 `100%` 全身、`350%` 琴头 / 指板和 `350%` 桥尾 / 控制区三类静态渲染复验；与原四把同框时，识别部件密度和材料层次不得出现明显断档。

首版资产系统至少包含：

- 4 种基础琴体语言：板式 single-cut、雕面 single-cut、offset 与十四品接柄 Dreadnought 空心木吉他。
- 4 种主体材质：桃花心木、枫木、沼泽白蜡木、半透明黑色树脂。
- 3 种电吉他拾音器布局：双线圈、单单双、双单线圈；木吉他不使用可见电磁拾音器，以音孔、箱体容积、面板与音梁表达声学结构。
- 3 种琴桥：固定式、双摇式、复古颤音式。
- 3 种表面处理：高光漆、开放木蜡、轻度做旧。
- 2 种稀有超现实部件：矿物晶体护板、液态金属拾音器罩。

首个木吉他基准必须是可辨识的十四品 Dreadnought：`25.4 in` 有效弦长、20 品、云杉面板、桃花心木背侧板、X 音梁文化语义、圆形音孔与多圈 rosette、3+3 实心琴头、乌木指板与 belly bridge、补偿弦枕及六枚弦钉。它应强调宽下腹、较深箱体、强投射和低频空气共鸣，不能只是把电吉他换成木色或删除拾音器。

所有 SVG 模块共享统一画布、琴颈轴线、连接锚点、比例和左上方主光源。琴体木纹、漆面反射、金属拉丝、塑料护板、阴影、边缘磨损与弦的高光分别成层；模块更换后，接触阴影与遮挡关系必须保持正确。

内容验证阶段必须让“模块化”在界面上可见，而不只存在于代码目录中。首个共享插槽平台固定为 `25.5 in` bolt-on 电吉他，至少提供 3 个琴体壳、2 个琴颈、4 个拾音器 / 电路包、4 个琴桥包和 4 种饰面；第二个共享插槽平台固定为 `24.75 in` set-neck 电吉他，至少提供雕面单切、半空心双切、薄双角三种琴体，圆点 / 梯形两种包边指板，双封闭线圈、双皂条单线圈、迷你双线圈三种电路，止弦尾件、梯形尾件、短板颤音三种桥尾和 4 种饰面。评审者可以先选择结构平台，再逐类换件，并立即看到同一把琴重新装配。每个平台同时提供不少于 6 个由这些部件真实生成的经典文化变体，用来验证模块组合并非换色。

玩家正式配琴时，吉他与当前可选部件必须处于同一个视口，不允许先看完整吉他、再向下滚动才能找到选项。手机竖屏使用上方 `55–62dvh` 琴盒舞台与下方 `38–45dvh` 工单；桌面也保持上琴下单或居中单一仪式构图，不恢复常驻后台控制台。五类零件拆成五个顺序页面，每页只展示当前包裹和 `2–3` 个候选，页面本身不因一次配琴操作而滚动。

标本入口、制琴台、音序器、收藏与公共琴墙应属于同一个声音标本系统：暖白纸面、石墨黑细线、Signal Green 选择反馈、Rust 索引线、瑞士式无衬线与等宽编号。入口舞台只能呈现灰银工程轮廓，不能提前展示实体成琴；标题、一个短说明和主按钮位于同一单屏，按钮文案必须清楚承诺“开始一份新标本”。界面不再模拟收据、印章、手写批注、琴盒、皮革或金属控制台；真实材质只存在于已装配的吉他模块。完整色彩、排版、材质、图标和动效系统见 `doc/visual.md`。

## 3. Game Mechanics

### 3.1 制琴流程

- 一次完整制琴是 5 个独立页面：琴体、琴颈、拾音器、琴桥、饰面。结构平台在开工单时由系统随机抽取，玩家这一局只能在该平台内完成吉他。
- 每个步骤先显示一个未开启零件箱；玩家开启后只获得 `2–3` 张候选卡，选择并装配其中 `1` 张后才能进入下一页。页面不显示其他步骤的完整选项。
- 点击候选卡立即生成临时试装配置并更新同屏主吉他；临时配置不改变已确认配置、等级记录和后续兼容池，直到玩家点击确认。
- 琴体步骤固定抽出 3 个不同琴体；琴颈步骤抽出 2 个不同琴颈；拾音器、琴桥和饰面从当前兼容池中抽出最多 3 个不同模块。兼容池只有 1 个结构模块时，允许以普通 / 做旧 / 原型三种制造等级形成 2–3 张不同卡，但装配锚点保持相同。
- 每张候选卡独立抽取制造等级：Workshop 普通 `68%`、Select 稀有 `27%`、Archive 典藏 `5%`。等级改变收藏稀有度、最终音色微调和揭晓反馈，不改变物理兼容规则。
- 单局不提供免费刷新，也不允许返回上一页重选；玩家必须在当前开出的 2–3 张卡中完成取舍。新一局会生成新的工单 Seed、结构平台、候选顺序与制造等级。
- 玩家可在每步装配后试听一次当前半成品；更换影响音色的部件后，音色指纹在 180 毫秒内过渡到新值。
- 五个步骤完成后先进入全屏调音步骤；玩家可对 Clean / Drive 做即时 A/B 试听，或从该局已揭示的候选中重新选择一个部件。确认后再生成永久 Guitar ID、收藏编号和 6 维音色指纹：温暖度、明亮度、起音、延音、驱动感、空间感，数值范围均为 0–100。
- 调音页还提供 4 块可串联效果器：Boost、Overdrive、Chorus、Tape Echo。点选单块立即加入或移出当前链路，并用同一 `2.4 秒` 比较句试听；新增单块追加到链末。效果器链随发布作品保存，并在 Remix 时一并带入新草稿。
- 相同 Guitar ID 与相同部件组合必须始终产生相同音色；重新打开、发布或 Remix 不改变原琴音色。
- 抽取不消耗付费货币；所有概率在开箱说明中公开。超现实模块后续进入 Archive 卡池，但仍必须服从连接、受力和弦路规则。
- 玩家首次进入拥有 6 个琴位，完成 3 把吉他后扩展到 12 个；首版个人收藏上限为 24 把。

### 3.1.1 模块插槽与兼容性

- 首个 `25.5 in bolt-on` 插槽固定六个结构锚点：琴枕、琴颈袋中心、琴桥弦鞍线、左右外弦、控制腔和琴身底部；更换模块时有效弦长误差不得超过画布 `±1.5 units`。
- 可见装配顺序固定为琴身侧板 → 琴身面板 / 饰面 → 护板 → 琴颈 → 拾音器与电路 → 琴桥 → 琴弦；任何模块不得靠独立缩放补偿接口错误。
- 首轮琴体壳为板式单切、轮廓双切和偏移琴体；琴颈为一体枫木六联与深色指板六联；电路包为双单线圈、SSS、HSS 与双宽单线圈；琴桥包为三鞍桥板、同步颤音、六鞍硬尾与浮动颤音。
- 兼容性不是完全自由随机：三鞍桥板只装在板式单切，浮动颤音只装在偏移琴体；同步颤音只装在轮廓双切；六鞍硬尾作为三个琴体的通用回退。选择不兼容模块时，界面必须禁用并显示“接口不匹配”，不能静默拼出物理错误。
- 评审预设至少覆盖：五十年代板式单切、五十年代轮廓双切、七十年代 HSS 热改、六十年代偏移颤音、双线圈板式改装和硬尾偏移实验。预设名称去品牌化，但结构文化必须能在缩略图中辨认。
- 第二个 `24.75 in set-neck` 插槽与 bolt-on 平台完全隔离：有效弦长为 `628.65 mm`，22 品，3+3 琴头，琴颈接入、桥位和尾件锚点使用独立坐标；界面不得把六联 bolt-on 琴颈装到 set-neck 琴体上。
- Set-neck 琴体为雕面单切、中央实心半空心双切和薄双角；圆点指板用于半空心语言，梯形指板可用于三类实心 / 半空心变体。拾音器包为双封闭线圈、双皂条单线圈和双迷你双线圈。
- Set-neck 兼容规则：梯形尾件只允许半空心；短板颤音只允许薄双角；止弦尾件作为三类琴体的通用回退。迷你双线圈不装在薄双角大护板上；选择琴体后若当前桥尾或拾音器失配，系统自动切到该琴体的第一个兼容件并显示更新后的清单。
- Set-neck 评审预设至少覆盖五十年代雕面双线圈、金色皂条单切、红色中心块半空心、自然色梯形尾件半空心、六十年代薄双角和短板颤音双角实验。预设必须来自实时模块，不保存整琴截图。

### 3.2 试琴室

- 六根弦采用标准定弦 E2、A2、D3、G3、B3、E4。
- 点按一根弦触发单次拨弦；在 220 毫秒内连续跨过至少 3 根弦判定为扫弦。
- 按住弦超过 350 毫秒进入延音预览，抬起后按该琴的延音参数自然衰减。
- Clean 与 Drive 两个放大器通道可以实时切换；切换采用 120 毫秒交叉过渡，避免爆音。
- 标准试听句为 4 小节、8 秒、120 BPM，所有吉他使用相同音符与力度，便于直接比较音色。
- A/B 比较最多固定两把琴，并使用同一试听句、同一通道与同一输出音量。

### 3.3 Riff Lab 音序器

- 音序器包含 6 条琴弦轨道、16 个时间步和最多 4 个连续小节。
- 手机端把 16 步拆成 4 个四拍小节页，一次只显示当前小节的 4 列；用四个安静的罗马数字切换小节。每格至少 `44×44 px`，页面和轨道都不发生横向溢出，也不让玩家猜测右侧还有被裁掉的内容。
- 每个格子拥有 4 个演奏状态：关闭、普通拨弦、重音、闷音；短按依次循环，长按 420 毫秒打开 0–12 品的音高选择。
- 默认速度为 120 BPM；当前成品提供 90 / 120 / 150 BPM 三档，后续高级模式再开放 70–160 BPM 单位步进。
- 默认拍号为 4/4，量化精度为十六分音符；首版不开放拍号切换。
- 完成制琴时从 6 条预载 First Riff（Dusty Eighths、Garage Pulse、Midnight Chime、Muted Engine、Slow Burn、Offset Glow）中随机附带 1 条，避免空白起步；进入 Riff Lab 后可一键随机换成其余预载节奏，再继续逐格编辑。
- 播放头按当前 BPM 循环；停止后回到当前小节第 1 步，暂停则保留位置。
- 播放期间编辑格子不需要停止或重新播放：新增、关闭、重音与闷音必须在播放头下一次到达该格时立即使用最新状态；切换 BPM 从下一步间隔开始生效。
- 提供 6 个可编辑起点：空白、直八摇滚、慢速分解、车库摇滚、梦幻延迟、闷音律动。
- “和弦印章”一次向所选 4 个时间步写入一个和弦，首版提供 C、G、Am、Em、F、D 六个和弦。
- 直接演奏录制最长 16 秒，结束后量化到最近的十六分音符；量化前后的结果均可撤销一次。
- 撤销历史保留最近 32 次编辑；切换页面或打开弹层不清空历史。
- 每把琴可以保存 8 段本地 Riff，其中 1 段可以设为公开 First Riff。

### 3.4 收藏、发布与公共琴墙

- 一条公开作品包含唯一 Entry ID、Guitar ID、吉他名称、模块配置、音色指纹、SVG 预览、First Riff、作者、发布时间和来源关系。
- 发布后立即进入玩家自己的琴墙视图，不等待远端同步完成。
- 公共琴墙默认展示最近 60 条作品，并支持按最新、最多试听、最多 Remix 三种顺序浏览。
- 点击作者头像与名字打开其 Aigram 主页；玩家自己的条目显示“你 / YOU”，不重复显示自己的头像。
- 点击作品主体进入详情；点击播放只试听，不进入详情。
- 琴墙详情必须是独立检视页，不复用成琴结算页。详情至少包含可缩放吉他、作者头像与名称、Guitar ID、五类部件、六维音色、First Riff 图样、BPM、播放 / 停止以及返回琴墙。
- Remix 复制源作品的 Riff 到新的草稿，但保持源 Guitar ID 与作者信息不可修改；Remix 发布后记录 `sourceEntryId`。
- Remix 首版只允许改编乐曲，不克隆或篡改原作者吉他的收藏归属。
- 同一玩家可以发布多把琴和同一把琴的多首作品；墙按 Entry ID 去重，不按用户去重。

### 3.5 反馈与进展

- 开箱输入在同一帧让箱体进入开启态；卡片在 `420` 毫秒内按 70 毫秒间隔依次出现。Workshop / Select / Archive 分别使用 1 / 2 / 3 级反馈强度。
- 完成每个部件选择时提供 180–260 毫秒的装配反馈和一次对应材质音，并把五步进度推进 1 格。
- 成琴揭晓持续 1.8 秒：工作灯亮起、琴体高光从琴头扫至琴桥、六根弦依次轻响。
- 首次完成制琴、首次完成 Riff、首次发布和首次被 Remix 分别形成独立里程碑。
- 游戏不使用传统分数；收藏进展以已造琴数量、音色范围覆盖、作品数量和被 Remix 次数呈现。

## 4. Controls

- 轻触部件卡：选择并立即装配；再次轻触已选部件播放其差异试听。
- 轻触“琴体 / 琴颈 / 拾音器 / 琴桥 / 饰面”分类：只切换当前零件托盘，不移动或遮挡吉他。
- 轻触“零件 / 配方”模式：在单件装配与六个文化预设之间切换；两种模式共享同一琴架和结构平台。
- 左右拖动吉他：在不超过正负 8 度的范围内查看表面反光与厚度暗示；抬起后 260 毫秒回正。
- 轻触琴弦：拨奏单弦；连续划过至少 3 根琴弦：按经过顺序扫弦。
- 轻触音序器格：循环关闭、普通、重音、闷音状态。
- 长按音序器格 420 毫秒：打开品位选择；拖到 0–12 后抬起确认。
- 点击 Play：从当前小节第 1 步播放；播放中点击切换为 Pause。
- 点击 Stop：停止并回到当前小节第 1 步。
- 键盘辅助：Space 播放或暂停，Escape 关闭弹层，方向键移动格子焦点，Enter 修改格子状态，数字 1–6 试听六根弦。
- 公共琴墙中的卡片和行使用 Click 激活，不能在 Pointer Down 时打开，确保触摸滚动不被误触中断。

## 5. Win / Lose Conditions

本游戏没有传统死亡或失败状态。一次创作在同时满足以下条件时视为完成：

1. 五个制琴步骤均已确认。
2. 吉他已命名，名称长度为 1–28 个字符。
3. 至少完成 1 小节 Riff，且其中包含至少 4 个启用的音符格。
4. 玩家选择保存为私人作品或发布到公共琴墙。

完成页首先只展示完整吉他、收藏编号和一个主要去向“写一段 First Riff”；等级明细折叠进制造档案，标准试听、收入琴架与再做一把降为次级动作，不再以四个同权按钮同时争夺注意力。发布失败时保留全部本地草稿，并提供重试与返回琴房两个入口；无声音权限时允许继续造琴和编辑，但完成前明确提示用户尚未试听。

## 6. Sound Effects

### 吉他声音

- 六根弦的基础音高为 E2 82.41 Hz、A2 110.00 Hz、D3 146.83 Hz、G3 196.00 Hz、B3 246.94 Hz、E4 329.63 Hz。
- 普通拨弦起音为 4–14 毫秒，主体衰减为 0.8–4.5 秒，最终值由音色指纹决定。
- 重音相对普通拨弦提升 3–5 dB，并增加 12–20 毫秒的明亮起音成分。
- 闷音的主体衰减限制在 90–220 毫秒，低频冲击保持 40–90 毫秒。
- 扫弦逐弦间隔为 18–42 毫秒，速度由手势实际经过六根弦的时间决定。
- 输出总线限制峰值不高于 -1 dBFS；连续触发时优先保持节奏清晰，不能产生数字削波。

### 界面与制琴反馈

- 木制部件装配：90–140 Hz 的短促木质敲击，时长 70–110 毫秒，快速衰减。
- 金属部件装配：1.8–3.2 kHz 的轻微金属扣合声，时长 45–90 毫秒，音量低于吉他拨弦 8 dB。
- 旋钮调整：每个刻度一次 1.2–1.8 kHz 微弱机械 click，时长 18–28 毫秒。
- 格子开启：当前弦对应音高的 65 毫秒预听；关闭格子：600–900 Hz、35 毫秒的软按键声。
- 播放开始：两次 1.0 kHz 木块预拍，间隔由当前 BPM 决定；可在设置中关闭。
- 保存成功：C5–E5–G5 三音上行，单音 90 毫秒，总时长 320 毫秒。
- 发布成功：First Riff 最后一个和弦的 600 毫秒缩短回放，叠加一次低于主音量 10 dB 的琴盒锁扣声。
- 错误提示：220 Hz 的双脉冲，每次 80 毫秒，中间间隔 60 毫秒；禁止使用刺耳蜂鸣。
- 所有非吉他界面音效可独立关闭；系统静音或减少动态效果不会影响视觉状态提示。
