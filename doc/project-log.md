# Tone Foundry — 项目记录

## 当前基线（2026-07-21）

- 仓库：`yinxinghuan/tone-foundry`
- 线上：<https://yinxinghuan.github.io/tone-foundry/>
- 内部评审：<https://yinxinghuan.github.io/tone-foundry/review.html>
- 正式玩家循环：随机电吉他工单 → 五步限量选件 → 全屏调音 / 效果器链 → First Riff → 收藏 / 发布 → 公共琴墙 / 点赞 / Remix。
- 正式随机池：`25.5 in bolt-on` 与 `24.75 in set-neck`，共 11 个可玩电吉他琴体。
- 原声评审池：`25.4 in acoustic flat-top`，含 Grand Concert、Grand Auditorium、Dreadnought、Super Jumbo 四种箱体；用于自由拼装、视觉校准和可拨弦母版验证，尚未接入正式随机工单。

## 当前验收结论

- `npm run build` 与严格 UI 审计在 `65962ea` 前通过。
- GitHub Actions 已通过：`65962ea` 对应部署工作流成功。
- 公共墙展示的是实时 SVG 配置，不保存或依赖整琴位图；因此生产 SVG 的细节修复会覆盖历史墙条目。
- 静态原声材质校验表明 4 种箱体比例和 4 种高对比饰面可分辨；真实线上交互画面仍以玩家在手机端的检视反馈为最终依据。

## 下一阶段建议顺序

1. 把原声平台引入正式 `BuildPlatform`、候选抽取、`buildTone` 与成琴保存，保持和电平台相同的 2–3 件限量抽取规则。
2. 逐把复核校准母版的琴头、指板末端、琴桥与面板木纹；每把至少做全身、琴头 / 指板、桥尾 / 控制区三种静态检查。
3. 用真实手机截图复验公共墙的 `160 × 160` 缩略图、详情缩放及中英文长标题，优先修复任何看不出弦钮、琴型或桥型的卡片。
4. 再扩展原声音色参数与预载 Riff：箱体体积、面板材质、传感方案和效果器链应共同影响试听和推荐 Riff。
